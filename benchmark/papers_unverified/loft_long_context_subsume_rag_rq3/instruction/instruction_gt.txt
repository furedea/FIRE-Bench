## Evaluation Details

- **Metric Definition**: 
  - Classification accuracy is defined as the number of correct predictions divided by the total number of predictions. Use the formula: 
    \[
    \text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Total Number of Predictions}}
    \]
- **Normalization/Pre-processing**: 
  - Ensure that the model outputs are parsed correctly to extract the predicted class labels. This may involve stripping any extraneous text or formatting from the model's output.
- **Aggregation Strategy**: 
  - Calculate accuracy per dataset and then average across all datasets to obtain an overall performance measure for the many-shot ICL tasks.

## Implementation Notes

- **Prompt Design**: 
  - Use the Corpus-in-Context (CiC) prompting strategy, which involves providing a large number of examples in the context before the query.
  - Ensure that the prompt includes clear instructions and examples formatted consistently to guide the model's output.
- **Order of Operations**: 
  - First, prepare the input context with examples, then append the query, and finally, process the model's output to extract predictions.
- **Pitfalls**: 
  - Be cautious of the model's tendency to rely on parametric knowledge rather than the provided context. Ensure that the context is comprehensive and relevant to the query.
- **Special Considerations**: 
  - For tasks with complex reasoning requirements, consider including chain-of-thought reasoning in the examples to improve model performance. However, note that this may not always lead to improvements, as observed with GPT-4o.
