## Experimental Design

### 1. Retrieval-Augmented Generation (RAG) Performance Evaluation
- **Procedure**:
  - Use the datasets NQ, TopiOCQA, HotPotQA, MuSiQue, QAMPARI, and QUEST.
  - For each dataset, construct a shared corpus by including all gold passages from few-shot, development, and test queries, then sample additional passages uniformly until reaching the desired context size (32k, 128k, or 1M tokens).
  - Shuffle gold and random passages to avoid positional biases.
  - Use the Gemini 1.5 Pro model to perform RAG by reasoning over the entire corpus in context.
  - Compare against a specialized retrieve-and-read RAG pipeline using Gecko to retrieve the top-40 documents, which are then used by Gemini 1.5 Pro to generate answers.

## Evaluation Details

- **Metric Definitions**:
  - **Subspan Exact Match**: For each dataset, compute the exact match between the predicted and gold answers at the subspan level. For multi-answer datasets like QAMPARI and QUEST, use a linear sum assignment algorithm to match predicted answers to gold answers based on overlap.
  - **MRecall@k**: For retrieval tasks with multiple gold targets, use MRecall@k, where k is dataset-specific (e.g., k=2 for HotPotQA).

- **Normalization and Pre-processing**:
  - Ensure that model outputs are tokenized consistently with the gold answers using the SentencePiece tokenizer.
  - For multi-answer datasets, align predicted and gold answers before computing exact match scores.

- **Aggregation Strategy**:
  - Compute per-sample subspan exact match scores.
  - Aggregate scores to obtain per-dataset performance.
  - Compare aggregated scores across different context lengths (32k, 128k, 1M) and against the RAG pipeline.

## Implementation Notes

- **Prompt Design**:
  - Use Corpus-in-Context (CiC) prompting, which includes task-specific instructions, a shared corpus, few-shot examples, and a query.
  - Ensure that the query is placed at the end of the prompt to leverage prefix-caching in autoregressive models.
  - Use Chain-of-Thought reasoning in few-shot examples to improve performance on multi-hop reasoning tasks.

- **Order of Operations**:
  - Construct the corpus by first including all gold passages, then sampling additional passages.
  - Shuffle passages to avoid positional biases before feeding them into the model.
  - Evaluate models on the maximum context size that fits without truncating the corpus or examples.
