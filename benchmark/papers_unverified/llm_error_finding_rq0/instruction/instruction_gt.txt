## Experimental Design

### 1. Mistake Finding in CoT Traces
- **Procedure**: 
  - Use the BIG-Bench Mistake dataset, which contains 2186 CoT-style traces annotated with the location of the first logical mistake.
  - Implement three prompting methods for mistake finding:
    1. **Direct Trace-Level Prompting**: Provide the entire trace to the model and prompt for the mistake location. The model outputs either the step number or "No" if no mistake is found.
    2. **Direct Step-Level Prompting**: For each step, prompt the model with the partial trace up to that step, asking for a binary Yes/No output indicating correctness. The first "No" indicates the mistake location.
    3. **CoT Step-Level Prompting**: Similar to direct step-level, but the model generates a series of reasoning steps for each partial trace. The first "No" indicates the mistake location.
  - Use temperature = 0 for all generations.
  - Evaluate across five tasks: word sorting, tracking shuffled objects, logical deduction, multi-step arithmetic, and Dyck languages.
- **What to Record**: 
  - Mistake finding accuracy for each model and prompting method.
  - Separate accuracy for traces with and without mistakes.
- **Correspondence**: Section 3, Table 4, and Table 9.

### 2. Correction of Reasoning Mistakes
- **Procedure**: 
  - Use oracle mistake location information from the BIG-Bench Mistake dataset.
  - Implement a backtracking method:
    1. Generate an initial CoT trace with temperature = 0.
    2. Identify the mistake location using oracle labels.
    3. If a mistake is found, regenerate the step at temperature = 1 until a different step is produced (up to 8 attempts).
    4. Replace the original step with the new one and regenerate the remaining steps at temperature = 0.
  - Compare results with a random baseline where mistake locations are randomly selected.
- **What to Record**: 
  - Changes in accuracy for correct and incorrect original traces.
  - Compare results with random baseline.
- **Correspondence**: Section 4, Table 6.

### 3. Classifier-Based Mistake Finding
- **Procedure**:

  - Evaluate the classifier's mistake-finding accuracy on held-out tasks.
  - Simulate classifiers at different accuracy levels to determine the minimum accuracy required for effective backtracking.
- **What to Record**:

## Evaluation Details

- **Mistake Finding Accuracy**: 
  - Defined as the percentage of traces where the model correctly identifies the mistake location or correctly identifies that there are no mistakes.
  - Separate accuracy calculations for traces with mistakes (incorrectmis) and without mistakes (correctmis).

- **Correction Accuracy**: 
  - Measured as the change in accuracy of the final answer (accuracyans) before and after backtracking.
  - Separate calculations for originally correct (correctans) and incorrect (incorrectans) traces.

- **Classifier Evaluation**: 
  - Mistake finding accuracy of the classifier is compared to the baseline of few-shot prompting.
  - Evaluate the impact of classifier accuracy on the effectiveness of backtracking.
