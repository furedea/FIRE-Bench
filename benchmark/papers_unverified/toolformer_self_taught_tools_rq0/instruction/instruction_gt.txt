## Experimental Design

### 1. Zero-Shot Performance on LAMA Benchmark Subsets
- **Procedure**: Evaluate the model on the SQuAD, Google-RE, and T-REx subsets of the LAMA benchmark. Filter out examples where the mask token is not the final token. Use a zero-shot setup without the Wikipedia Search API.
- **Input Format**: Provide the model with a statement missing a fact and prompt it to complete the statement.
- **Prompting Strategy**: Use the prompt "Please complete the following text so that it is factually correct: [statement]."
- **Number of Samples**: Use the entire filtered dataset.
- **What to Record**: Record the top five predicted words for each example.
- **Correspondence**: Results correspond to Table 3 in the paper.

### 2. Mathematical Reasoning on ASDiv, SVAMP, and MAWPS
- **Procedure**: Evaluate the model's mathematical reasoning abilities using the ASDiv, SVAMP, and MAWPS datasets. Use a zero-shot setup with the calculator tool.
- **Input Format**: Provide the model with a context and a question.
- **Prompting Strategy**: Use the prompt "[context] [question] The answer is."
- **Number of Samples**: Use the entire dataset for each benchmark.
- **What to Record**: Record the first number predicted by the model, or the first number after the "=" sign if the prediction contains an equation.
- **Correspondence**: Results correspond to Table 4 in the paper.

### 3. Question Answering on Web Questions, Natural Questions, and TriviaQA
- **Procedure**: Evaluate the model on question answering datasets using a zero-shot setup with the Wikipedia search tool.
- **Input Format**: Provide the model with a question.
- **Prompting Strategy**: Prepend the question with "Answer the following question:" and ensure it ends with a question mark.
- **Number of Samples**: Use the entire dataset for each benchmark.
- **What to Record**: Record whether the first 20 words predicted by the model contain the correct answer.
- **Correspondence**: Results correspond to Table 5 in the paper.

### 4. Multilingual Question Answering on MLQA
- **Procedure**: Evaluate the model on the MLQA benchmark using a zero-shot setup with the machine translation tool.
- **Input Format**: Provide the model with a context paragraph in English and a question in one of the target languages.
- **Prompting Strategy**: Use the prompt "Your task is to answer a question based on the following paragraph: [context] Now answer the following question in English: [question]."
- **Number of Samples**: Use the entire dataset for each language.
- **What to Record**: Record the percentage of times the model's generation, capped at 10 words, contains the correct answer.
- **Correspondence**: Results correspond to Table 6 in the paper.

### 5. Temporal Reasoning on TEMPLAMA and DATESET
- **Procedure**: Evaluate the model's temporal reasoning abilities using TEMPLAMA and DATESET. Use a zero-shot setup with the calendar tool.
- **Input Format**: Provide the model with a cloze query or a temporal question.
- **Prompting Strategy**: Use the same prompt as for LAMA.
- **Number of Samples**: Use the entire dataset for each benchmark.
- **What to Record**: Record whether the correct answer is within the first five words predicted by the model.
- **Correspondence**: Results correspond to Table 7 in the paper.

## Evaluation Details

- **Metric Definitions**: 
  - **Accuracy**: For LAMA, check if the correct word is within the first five predicted words. For math datasets, check if the first predicted number matches the expected answer. For QA datasets, check if the correct answer is within the first 20 predicted words.
  - **Perplexity**: Evaluate on WikiText and a subset of CCNet to ensure language modeling performance is not degraded.

- **Normalization and Pre-processing**: 
  - For math datasets, equations in predictions should be parsed to extract the final numerical answer.
  - For QA datasets, ensure questions are properly formatted with a question mark.

- **Aggregation Strategy**: 
  - Compute per-sample accuracy, then aggregate to obtain per-condition accuracy (e.g., per dataset or per language), and finally compute overall accuracy across all conditions.

## Implementation Notes

- **Output Parsing**: For math datasets, parse equations to extract numerical answers. For QA datasets, check the presence of the correct answer within a specified word limit.
- **Order of Operations**: Ensure API calls are made before continuing with token prediction, as described in the inference section of the paper.
