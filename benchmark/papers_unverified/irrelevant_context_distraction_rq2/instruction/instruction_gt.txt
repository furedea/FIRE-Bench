## Experimental Design

### 1. Analyze Performance Based on Factors of Irrelevant Context
- **Procedure**: 
  - Use the GSM-IC dataset, which includes problems with irrelevant context added to the original GSM8K problems.
  - For each problem, generate variations by adding irrelevant sentences characterized by three factors: topic (in-topic vs. off-topic), role name overlap (yes vs. no), and number range (in-range vs. out-of-range).
  - Evaluate the models using these variations to determine how each factor affects performance.
  - Use code-davinci-002 and text-davinci-003 models with different prompting techniques: Chain-of-Thought (COT), Least-to-Most (LTM), and Program (PROGRAM).
- **What to Record**: 
  - Micro accuracy: Average accuracy over all test problems.
  - Macro accuracy: Average accuracy over classes of test problems, where each class consists of test examples derived from the same base problem.
  - Normalized accuracy: Micro or macro accuracy normalized by the base problem accuracy.
- **Correspondence**:

## Evaluation Details

- **Metric Definitions**:
  - **Micro Accuracy**: 
    \[
    \text{Acc}_{\text{micro}}(M; P) = \frac{\sum_{p \in P} \mathbb{1}[M(p) = s(p)]}{|P|}
    \]
    where \( M(p) \) is the model's prediction for problem \( p \), and \( s(p) \) is the standard solution.
  - **Macro Accuracy**: 
    \[
    \text{Acc}_{\text{macro}}(M; B) = \frac{\sum_{b \in B} \mathbb{1}[\bigwedge_{p \in P(b)} M(p) = s(p)]}{|B|}
    \]
    where \( P(b) \) is the set of test examples derived from base example \( b \).
  - **Normalized Accuracy**: 
    \[
    \text{norm}(a_M; M) = \frac{a_M}{n_M}
    \]
    where \( a_M \) is the micro or macro accuracy, and \( n_M \) is the base problem accuracy of method \( M \).

## Implementation Notes

- **Prompt Design**:
  - For few-shot prompting techniques (COT, LTM, PROGRAM), use one exemplar problem with its solution in the prompt. The exemplar can be based on either the original problem or the problem with irrelevant context.
  - For zero-shot chain-of-thought (0-COT), directly present the problem followed by "A: Let’s think step by step:".
  - Instructed prompting involves adding the sentence "Solve grade school math problems. Feel free to ignore irrelevant information given in the questions." before the exemplars in the prompt.
- **Order of Operations**:
  - Ensure that the irrelevant sentence does not affect the standard solution of the base problem by manually verifying each generated sentence.
  - Use greedy decoding for experiments without self-consistency and sample 20 responses with temperature \( \tau = 0.7 \) for self-consistency experiments.
