## Evaluation Details

### Shift Ciphers: Encoding Accuracy
- **Metric Definition**: 
  - Encoding accuracy is defined as the percentage of correctly encoded sentences out of the total number of sentences.
  - Use Levenshtein edit distance to measure the similarity between the model output and the correct encoded sentence.
- **Normalization**: 
  - Normalize the input sentences to ensure consistent tokenization across models.
- **Aggregation Strategy**: 
  - Calculate per-sample accuracy and edit distance, then aggregate these metrics per condition (high, medium, low probability) and overall.

### Counting Accuracy
- **Metric Definition**: 
  - Counting accuracy is defined as the percentage of correct counts provided by the model out of the total number of counting tasks.
- **Normalization**: 
  - Ensure that all input sequences are tokenized consistently and that the counting task is clearly defined in the prompt.
- **Aggregation Strategy**: 
  - Calculate per-sample accuracy, then aggregate these metrics per condition (high, low probability) and overall.

## Implementation Notes

### Shift Ciphers
- **Pitfalls**: 
  - Ensure that the input sentences are correctly categorized into probability levels to avoid skewed results.
  - Be cautious of the model's tendency to regularize low-probability outputs to higher-probability ones.
- **Prompt Templates**: 
  - Use a consistent prompt template that clearly instructs the model to encode the sentence into rot-13.
  - Include an example in the prompt to clarify the task.

### Counting
- **Pitfalls**: 
  - Ensure that the input sequences are free from ambiguities that might affect the counting task.
  - Be aware of the model's potential bias towards high-probability numbers.
- **Prompt Templates**: 
  - Use a clear and concise prompt that specifies the counting task and the format of the input sequence.
  - Include examples to illustrate the task.
