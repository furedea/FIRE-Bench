## Experimental Design

### 1. Prompt Language and Parameter Impact on Code Security
- **Procedure**: 
  - Participants will interact with an AI code assistant to solve five security-related programming tasks. Each task involves a different aspect of security: encryption/decryption, message signing, file path handling, SQL operations, and C string manipulation.
  - Participants will be allowed to use various prompting strategies, including specifying task instructions, providing function declarations, and adjusting model parameters like temperature and response length.
  - The order of tasks will be randomized for each participant to control for potential fatigue effects.
  - Participants will be divided into two groups: one with access to the AI assistant and one without (control group).
- **What to Record**: 
  - Record all prompts submitted to the AI assistant, including any changes in prompt language or parameters.
  - Capture the AI assistant's responses and whether participants accepted these responses (e.g., by copying them into their solution).
  - Log the final code submissions for each task, noting correctness and security classifications.
  - Collect participant feedback on their perceived correctness and security of their solutions, as well as their trust in the AI assistant.
- **Correspondence**: 
  - This experiment corresponds to the analysis in Section 6 of the paper, with results presented in Figure 4 and Table 6.

## Evaluation Details

- **Metric Definitions**:
  - **Security Classification**: Solutions will be classified as "Secure," "Partially Secure," or "Insecure" based on predefined criteria for each task. For example, in the encryption task, using a secure cipher and mode will classify the solution as "Secure."
  - **Prompt Strategy Analysis**: Prompts will be categorized into types such as "Specification," "Instruction," "Function Declaration," etc., and analyzed for their impact on the security of the resulting code.
- **Normalization and Pre-processing**:
  - Normalize prompt text by removing extraneous whitespace and standardizing function declarations for consistent analysis.
  - Pre-process code submissions to ensure they run without syntax errors before security evaluation.
- **Aggregation Strategy**:
  - Aggregate security classifications per task and participant to determine overall security trends.
  - Analyze prompt strategies and parameter adjustments per task to identify patterns associated with secure code.

## Implementation Notes

- **Key Pitfalls**:
  - The paper notes that participants often rely too heavily on AI-generated outputs without sufficient verification, leading to security vulnerabilities. Ensure participants are aware of the importance of verifying AI outputs.
- **Special Prompt Templates**:
  - Encourage participants to use specific prompt templates that include security-related language, such as "Ensure this code is secure" or "Use a secure library for encryption."
- **Order of Operations**:
  - Participants should first attempt to solve the task independently before consulting the AI assistant, to establish a baseline understanding of the problem.
- **Output Parsing Logic**:
  - Implement logic to parse AI outputs for common security issues, such as improper use of cryptographic libraries or SQL injection vulnerabilities, to assist in the manual coding process.
