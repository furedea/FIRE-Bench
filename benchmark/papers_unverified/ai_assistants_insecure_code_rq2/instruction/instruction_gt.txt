## Evaluation Details

- **Metric Definitions**:
  - **Security Classification**: 
    - Secure: No security mistakes.
    - Partially Secure: Minor security issues, such as using a bad source of randomness.
    - Insecure: Major security vulnerabilities, such as SQL injection or use of trivial ciphers.
  - **Prompt Categorization**: Based on the taxonomy provided in the paper, categorize each prompt into one or more types.
  - **Edit Distance**: Use normalized edit distance to measure similarity between user prompts and AI outputs.

- **Aggregation Strategy**:
  - Aggregate prompt types and security outcomes per user and per task.
  - Calculate the proportion of secure, partially secure, and insecure outcomes for each prompt type and parameter setting.
