- **Procedure**: 
  - For each task (multi-digit multiplication, logic grid puzzles, dynamic programming problem), generate computation graphs for both the ground truth and the model-generated outputs.
  - Use GPT-3 and GPT-4 models to generate outputs for each task.

  - Classify each node in the model-generated graph into one of four categories: fully correct, local error, propagation error, or restoration error.

- **What to Record**:

## Evaluation Details

- **Metric Definitions**:
  - **Local Error**: Occurs when a node's parent nodes have correct values, but the node itself is derived from an incorrect computation.
  - **Propagation Error**: Occurs when a node is derived from a correct computation, but some of its parent nodes have incorrect values.
  - **Restoration Error**: Occurs when a node has a correct value but is derived from an incorrect computation.
  - **Fully Correct**: Occurs when a node and all its ancestors have correct values and are derived from correct computations.

- **Normalization and Pre-processing**:
  - Ensure that all computation graphs are linearized into a topological order before analysis.
  - Normalize the error counts by the total number of nodes at each reasoning depth to facilitate comparison across different tasks and models.

- **Aggregation Strategy**:
  - Calculate the ratio of each error type per reasoning depth.
  - Aggregate these ratios to provide an overall error profile for each model and task.

## Implementation Notes

- **Pitfalls and Caveats**:
  - Ensure that the computation graph accurately reflects the sequence of operations required to solve each task.
  - Be cautious of potential biases introduced by the model's training data, which may affect error types and frequencies.

- **Special Prompt Templates**:
  - Use scratchpads to guide the model in generating intermediate steps for each task, as this can influence the types of errors observed.

- **Order of Operations**:
  - Maintain the sequence of operations as defined in the computation graph to ensure that error analysis is consistent with the task's logical structure.
