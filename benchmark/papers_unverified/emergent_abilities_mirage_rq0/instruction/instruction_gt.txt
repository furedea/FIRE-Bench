## Evaluation Details

- **Metric Definitions**:
  - **Accuracy**: Proportion of completely correct outputs (all tokens correct).
  - **Token Edit Distance**: Number of token substitutions, additions, or deletions needed to match the target sequence.
  - **Emergence Score**: Defined as `sign(arg max_i y_i - arg min_i y_i)(max_i y_i - min_i y_i) / sqrt(Median({(y_i - y_{i-1})^2}_i))`, where `y_i` is model performance at scale `x_i`.
  - **Brier Score**: Mean squared error between predicted probabilities and actual outcomes for binary tasks.

- **Normalization and Pre-processing**:
  - For Accuracy, ensure that outputs are tokenized consistently with the target sequences.
  - For Token Edit Distance, compute the distance for each output-target pair and average over the dataset.

- **Aggregation Strategy**:
  - Compute per-sample scores, then average to obtain per-condition scores.
  - Compare overall trends across different metrics and model scales.

## Implementation Notes

- **Pitfalls**:
  - Ensure sufficient test data to accurately measure performance, especially for nonlinear metrics like Accuracy.
  - Be cautious of overfitting when defining new metrics for vision tasks.

- **Special Prompt Templates**:
  - Use consistent input formatting for arithmetic tasks to ensure comparability across model scales.

- **Order of Operations**:
  - Collect raw outputs first, then apply different metrics to the same set of outputs to isolate the effect of metric choice.
