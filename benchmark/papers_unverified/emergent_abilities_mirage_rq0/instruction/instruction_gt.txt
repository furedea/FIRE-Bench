## Evaluation Details

- **Accuracy Metric**: 
  - Defined as the proportion of completely correct outputs (all tokens correct) over the total number of samples.
  - Computation: `Accuracy = (Number of completely correct outputs) / (Total number of outputs)`.
  - Edge Cases: Outputs with any incorrect token are scored as 0.

- **Token Edit Distance Metric**: 
  - Measures the number of token substitutions, additions, or deletions required to transform the model output into the correct output.
  - Computation: Use a library function to compute the Levenshtein distance adapted for tokens.
  - Normalize by dividing by the length of the target sequence to ensure comparability across different sequence lengths.

- **Aggregation Strategy**: 
  - For each metric, compute the average score across all samples for each model scale.
  - Compare the aggregated scores across different model scales to assess the presence or absence of emergent abilities.
