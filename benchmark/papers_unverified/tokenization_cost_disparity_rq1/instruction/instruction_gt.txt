## Experimental Design

### 1. Tokenization Premium Analysis
- **Procedure**: 
  - Use the FLORES-200 dataset to analyze tokenization lengths across different languages.
  - For each language pair, compute the tokenization premium using the formula: \(|t(sA)|/|t(sB)|\), where \(sA\) and \(sB\) are sentences in different languages, and \(|t(sA)|\) is the tokenized length.
  - Use subword tokenization models such as GPT-2, RoBERTa, and others mentioned in the paper.
  - Ensure that the analysis includes both English-centric and non-English-centric models.
- **What to Record**: 
  - Tokenization lengths for each language.
  - Tokenization premiums for each language pair.
  - Raw outputs of tokenized sentences.
- **Correspondence**: Section 4.1, 4.2, 4.3, and 4.4; Tables 1, 2, 3, 4, and 5.

### 2. Cost Implications Analysis
- **Procedure**: 
  - Calculate the cost premium for each language using the tokenization premiums obtained from the previous experiment.
  - Assume a pricing model where cost is proportional to the number of tokens.
- **What to Record**: 
  - Cost premiums for each language relative to English.
- **Correspondence**: Section 5.1.

### 3. Latency Implications Analysis
- **Procedure**: 
  - Measure the processing time for tokenized inputs using the RoBERTa model.
  - Plot the processing time against tokenization lengths to assess the correlation.
  - Conduct 20 independent runs for each sentence to ensure robustness.
- **What to Record**: 
  - Processing times for each language.
  - Correlation statistics between tokenization length and processing time.
- **Correspondence**: Figure 2; Section 5.2.

### 4. Long Context Processing Analysis
- **Procedure**: 
  - Compare the context processing capabilities of models like GPT-2 and GPT-4 across languages with varying tokenization premiums.
  - Analyze the maximum input length that can be processed for each language.
- **What to Record**: 
  - Maximum context length for each language.
  - Comparison of context processing capabilities.
- **Correspondence**: Section 5.3.

## Evaluation Details

- **Metric Definitions**:
  - **Tokenization Premium**: \(|t(sA)|/|t(sB)|\), where \(|t(sA)|\) is the tokenized length of sentence \(sA\).
  - **Cost Premium**: Directly proportional to the tokenization premium.
  - **Processing Time**: Measured in seconds, averaged over 20 runs.
  - **Context Capacity**: Maximum number of tokens that can be processed by the model.

- **Normalization and Pre-processing**:
  - Ensure all sentences are pre-tokenized using the same tokenizer settings.
  - Normalize tokenization lengths by dividing by the length of the English sentence for comparison.

- **Aggregation Strategy**:
  - Compute per-sample tokenization premiums and aggregate to obtain per-language averages.
  - Aggregate processing times across runs to obtain average latency per language.

## Implementation Notes

- **Pitfalls and Caveats**:
  - Ensure that the tokenizer used is consistent across all languages for a fair comparison.
  - Be aware of the potential for high variance in tokenization lengths due to language-specific characteristics.

- **Special Prompt Templates**:
  - Use consistent sentence structures across languages to minimize variability in tokenization.

- **Order of Operations**:
  - Perform tokenization before any other processing to ensure accurate measurement of tokenization lengths.
  - Conduct latency measurements after tokenization to capture the full processing pipeline.
