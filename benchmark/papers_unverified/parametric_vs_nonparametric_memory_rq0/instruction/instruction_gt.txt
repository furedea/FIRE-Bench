## Evaluation Details

- **Accuracy Metric**: 
  - A prediction is marked correct if any substring of the prediction is an exact match of any of the gold answers.
- **Correlation Calculation**: 
  - Calculate the correlation between log-transformed subject entity popularity and binary accuracy (correct/incorrect) for each question.
- **Aggregation Strategy**: 
  - Compute per-sample accuracy, then aggregate to per-condition (relationship type) and overall accuracy.

## Implementation Notes

- **Prompting Strategy**: 
  - Use a simple template for all questions: "Q: <question> A:".
  - Ensure consistency in few-shot examples by stratifying samples by relationship type for POPQA.
- **Retrieval Augmentation**: 
  - Use only the top retrieved paragraph to keep the input size manageable.
  - Be cautious of retrieval quality, as low-quality retrieval can mislead models.
- **Adaptive Retrieval**: 
  - Determine popularity thresholds using a development set and apply them independently for each relationship type.
  - Ensure that retrieval is only used when necessary to optimize both performance and efficiency.
