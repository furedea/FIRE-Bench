## Experimental Design

### 1. Position Bias Analysis
- **Procedure**: 
  - Use the MT-Bench dataset to generate responses from two models for each question.
  - Present these responses to GPT-4, GPT-3.5, and Claude-V1 in two different orders (i.e., swap the positions of the responses).
  - Use the default prompt for pairwise comparison as specified in Figure 5 of the Appendix.
  - Evaluate the consistency of the LLM judgments when the order of responses is swapped.
- **What to Record**: 
  - Consistency rate of judgments (percentage of cases where the judgment remains the same after swapping).
  - Instances of bias towards the first or second position.
  - Any errors in output format.
- **Correspondence**: Table 2, Section 3.3

### 2. Verbosity Bias Evaluation
- **Procedure**: 
  - Select 23 model answers from the MT-Bench dataset that contain a numbered list.
  - Create verbose versions of these answers by rephrasing the list without adding new information and inserting it at the beginning of the original list.
  - Present both the original and verbose responses to GPT-4, GPT-3.5, and Claude-V1 for evaluation.
- **What to Record**: 
  - Failure rate of LLM judges under the "repetitive list" attack (percentage of cases where the verbose response is incorrectly judged as better).
- **Correspondence**: Table 3, Section 3.3

### 3. Limited Capability in Grading Math and Reasoning Questions
- **Procedure**: 
  - Use math and reasoning questions from the MT-Bench dataset.
  - Evaluate responses using GPT-4 with and without reference answers.
  - Implement chain-of-thought and reference-guided grading methods as described in Figures 7 and 8 of the Appendix.
- **What to Record**: 
  - Failure rate of LLM judges in correctly grading math and reasoning questions.
  - Comparison of failure rates between default, chain-of-thought, and reference-guided prompts.
- **Correspondence**: Table 4, Section 3.3

## Evaluation Details

### Position Bias
- **Metric Definition**: Consistency rate is calculated as the percentage of cases where the LLM's judgment remains the same after swapping the order of responses.
- **Normalization**: Ensure that the responses are presented in a consistent format to avoid any influence from formatting differences.
- **Aggregation Strategy**: Calculate the overall consistency rate across all questions and model pairs.

### Verbosity Bias
- **Metric Definition**: Failure rate is defined as the percentage of cases where the verbose response is incorrectly judged as better.
- **Normalization**: Ensure that the verbose and original responses are presented in a similar format to avoid any influence from formatting differences.
- **Aggregation Strategy**: Calculate the overall failure rate across all questions.

### Limited Capability in Grading Math and Reasoning
- **Metric Definition**: Failure rate is defined as the percentage of cases where the LLM judge makes an incorrect judgment.
- **Normalization**: Use reference answers to provide a baseline for comparison.
- **Aggregation Strategy**: Calculate the failure rate for each prompt type (default, chain-of-thought, reference-guided) and compare.
