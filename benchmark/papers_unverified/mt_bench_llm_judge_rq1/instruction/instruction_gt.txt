## Experimental Design

### 1. Position Bias Analysis
- **Procedure**: 
  - Use the MT-bench dataset to generate pairs of responses for each question.
  - Present these pairs to the LLM judges (GPT-4, Claude-v1, GPT-3.5) in two orders: original and swapped.
  - Use the default prompt for pairwise comparison as shown in Figure 5 of the Appendix.
  - Record the consistency of judgments when the order of responses is swapped.
- **What to Record**: 
  - Consistency rate (percentage of cases where the judgment remains the same after swapping).
  - Bias towards the first or second position.
  - Error rate (incorrect output formats).

### 2. Verbosity Bias Evaluation
- **Procedure**: 
  - Select 23 model answers from MT-bench that contain a numbered list.
  - Create verbose versions of these answers by rephrasing and duplicating the list items.
  - Present both original and verbose answers to the LLM judges.
  - Use the default prompt for pairwise comparison.
- **What to Record**: 
  - Failure rate (percentage of cases where the verbose answer is incorrectly judged as better).

## Evaluation Details

### Position Bias
- **Metric Definitions**:
  - **Consistency**: Percentage of cases where the LLM judge gives the same result when the order of responses is swapped.
  - **Bias towards first/second**: Percentage of cases where the LLM judge favors the first or second response.
  - **Error Rate**: Percentage of cases with incorrect output formats.
- **Aggregation Strategy**: 
  - Calculate metrics per model and prompt type, then aggregate across all questions.

### Verbosity Bias
- **Metric Definitions**:
  - **Failure Rate**: Percentage of cases where the verbose response is judged as better.
- **Aggregation Strategy**: 
  - Calculate the failure rate for each LLM judge across all verbose test cases.
