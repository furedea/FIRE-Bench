## Experimental Design

### 1. LLM-based Choice Extraction Evaluation
- **Procedure**: 
  - For each question in the MMBench dataset, use the VLM to generate a prediction.
  - Attempt to extract the choice label from the VLM's prediction using heuristic matching.
  - If heuristic matching fails, use an LLM (e.g., GPT-4) to extract the choice label by providing the question, choices, and VLM prediction to the LLM and asking it to align the prediction with one of the given choices.
  - If the LLM finds that the model prediction is significantly different from all choices, it should return a pseudo choice 'Z'.
- **What to Record**: 
  - Record the success rate of heuristic matching.
  - Record the alignment rate between LLM-extracted choices and human annotations.
  - Record the final accuracy of VLM predictions after LLM-based choice extraction.

### 2. CircularEval Strategy Evaluation
- **Procedure**: 
  - For each question in the MMBench dataset, perform multiple inference passes with the VLM, each time applying a circular shift to the choices.
  - A VLM is considered successful in solving a question only if it correctly predicts the answer in all circular passes.
- **What to Record**:

## Evaluation Details

### LLM-based Choice Extraction
- **Metric Definition**: 
  - Alignment rate is defined as the percentage of cases where the LLM-extracted choice matches the human-annotated choice.
  - Use the `gpt-4-0125` model for choice extraction due to its superior alignment capability.
- **Normalization/Pre-processing**:

- **Aggregation Strategy**: 
  - Calculate the alignment rate per sample, then aggregate to obtain the overall alignment rate.

### CircularEval Strategy
- **Metric Definition**: 
  - Accuracy is defined as the percentage of questions correctly answered in all circular passes.
- **Normalization/Pre-processing**: 
  - Apply circular shifts to the choices for each inference pass.
- **Aggregation Strategy**: 
  - Calculate accuracy per sample, then aggregate to obtain the overall accuracy.
