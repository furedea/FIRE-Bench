## Evaluation Details

- **Metrics**:
  - **F1 Score**: Used for QA tasks, calculated as the harmonic mean of precision and recall.
  - **ROUGE-L**: Used for summarization tasks, measures the longest common subsequence between the generated and reference summaries.
  - **Accuracy**: Used for classification tasks, calculated as the number of correct predictions divided by the total number of predictions.
  - **Edit Sim**: Used for code completion tasks, based on Levenshtein distance.

- **Normalization and Pre-processing**:
  - For few-shot learning and code completion tasks, extract the first line of the response.
  - For code completion, ignore comment lines in the output.

- **Aggregation Strategy**:
  - Compute per-sample scores, then average to obtain per-condition scores.
  - Report macro-average scores across all tasks for overall performance.

## Implementation Notes

- **Prompt Templates**: 
  - Specific templates are provided for each task, ensuring the models understand the task requirements.
  - Instructions are placed at both the beginning and end of the prompt to reinforce task understanding.

- **Truncation Strategy**: 
  - When input length exceeds the model's maximum context length, truncate from the middle to preserve crucial information at the start and end of the sequence.

- **Order of Operations**:
  - Ensure that the input format and maximum output length are set before model evaluation.
  - Apply context compression techniques before feeding inputs to the models.
