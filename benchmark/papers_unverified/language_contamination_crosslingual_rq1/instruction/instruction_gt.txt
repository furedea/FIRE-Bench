## Evaluation Details

- **Bits per Character (BPC)**: 
  - Calculate BPC by dividing the total number of bits required to encode the text by the number of characters in the text.
  - Use this metric to compare models with different tokenizations.

- **Accuracy for POS Tagging**: 
  - Compute the percentage of correctly predicted POS tags out of the total number of tags.
  - Evaluate both probing and fine-tuning setups.

- **Spearman Correlation**: 
  - Use the `scipy.stats.spearmanr` function to compute the Spearman correlation coefficients.
  - Consider p-values to determine statistical significance, with thresholds at p < 0.05 and p < 0.001.

## Implementation Notes

- **Tokenization**: 
  - Be aware of the tokenization differences across models, especially the handling of non-Latin scripts and the presence of UNK tokens.
  - RoBERTa uses byte-level BPE encoding, which avoids UNK tokens for Unicode text, unlike BERT and T5.

- **Probing and Fine-tuning Setup**: 
  - For probing, use a linear classifier with parameters `m * l`, where `m` is the encoder's output dimension and `l` is the label set size.
  - For fine-tuning, unfreeze the encoder weights to allow updates during training.

- **Batch Sizes and Learning Rates**: 
  - Use a batch size of 256 for frozen models and 16 for fine-tuned models.
  - Set learning rates to 0.001 for probing and 5e-6 for fine-tuning.
  - Use the Adam optimizer for training.

- **Hardware Requirements**: 
  - Conduct experiments on Nvidia V100 GPUs, with 16GB for frozen models and 32GB for fine-tuned models.
