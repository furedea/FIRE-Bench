## Evaluation Details

- **Metric Definitions**:
  - **Local Error**: Occurs when a node's computation is incorrect despite having correct inputs.
  - **Propagation Error**: Occurs when a node's computation is correct, but it receives incorrect inputs from its predecessors.
  - **Restoration Error**: Occurs when a node has a correct value despite being derived from incorrect computations.
- **Normalization/Pre-processing**:
  - Ensure all model outputs are tokenized consistently for comparison.
  - Align computation graph nodes with model output tokens for accurate error categorization.
- **Aggregation Strategy**:
  - Calculate the proportion of each error type per layer in the computation graph.
  - Aggregate error proportions across all samples to determine overall error distribution patterns.

## Implementation Notes

- **Pitfalls/Caveats**:
  - Ensure that the computation graph accurately reflects the model's reasoning process, as discrepancies can lead to misclassification of error types.
  - Be cautious of models' tendency to memorize rather than reason, which may manifest as restoration errors.
- **Special Prompt Templates**:
  - Use structured prompts that guide the model through step-by-step reasoning, especially for scratchpad-based approaches.
- **Order of Operations**:
  - Maintain a consistent order of operations in constructing computation graphs to ensure comparability across different models and tasks.
