## Experimental Design

### 1. Positional Bias Evaluation
- **Procedure**: 
  - Use the Vicuna Benchmark dataset, which includes 80 questions across 9 categories.
  - For each question, generate two responses using ChatGPT and Vicuna-13B.
  - Create two evaluation prompts for each question: one with the original order of responses and another with the responses swapped.
  - Use GPT-4 and ChatGPT as evaluators to score the responses in both orders.
  - Ensure that the evaluation template includes slots for the question and both responses, and instructs the evaluator to rate each response on a scale of 1 to 10.
- **What to Record**: 
  - Raw scores for each response in both positions.
  - Conflict Rate, calculated as the proportion of examples where the evaluation outcome changes when the order is swapped.
- **Correspondence**: 
  - This experiment corresponds to Table 2 and Section 2.2 in the paper.

### 2. Multiple Evidence Calibration (MEC)
- **Procedure**: 
  - Use the same setup as the positional bias evaluation.
  - For each evaluation prompt, generate multiple evaluation evidences (k = 3) before assigning scores.
  - Use a temperature setting of 1 to sample diverse evaluation evidences.
- **What to Record**: 
  - Multiple scores for each response based on different evidences.
  - Average scores across all evidences.
- **Correspondence**: 
  - This experiment corresponds to Section 3.1 and Figure 3(a) in the paper.

### 3. Balanced Position Calibration (BPC)
- **Procedure**: 
  - Combine MEC with BPC by evaluating each response in both positions and averaging the scores.
  - For each question, generate 2k scores by swapping positions and using MEC.
- **What to Record**: 
  - Final calibrated scores for each response, calculated as the average of 2k scores.
- **Correspondence**: 
  - This experiment corresponds to Section 3.2 and Figure 3(b) in the paper.

### 4. Human-in-the-Loop Calibration (HITLC)
- **Procedure**: 
  - Use the results from MEC and BPC to calculate the Balanced Position Diversity Entropy (BPDE) for each example.
  - Select the top-β% examples with the highest BPDE scores for human annotation.
  - Integrate human annotations into the final evaluation.
- **What to Record**: 
  - BPDE scores for each example.
  - Final evaluation results after incorporating human annotations.
- **Correspondence**: 
  - This experiment corresponds to Section 3.3 and Figure 3(c) in the paper.

## Evaluation Details

### Conflict Rate Calculation
- **Formula**: 
  \[
  \text{Conflict Rate} = \frac{\sum_{i=1}^{N} I(\text{ER}_{r1r2}^i \neq \text{ER}_{r2r1}^i)}{N}
  \]
  where \(I(.)\) is the indicator function, \(N\) is the total number of examples, and \(\text{ER}_{r1r2}^i\) and \(\text{ER}_{r2r1}^i\) are the evaluation results for the original and swapped positions, respectively.

### Balanced Position Diversity Entropy (BPDE)
- **Formula**: 
  \[
  \text{BPDE} = \sum_{er \in \{\text{win}, \text{tie}, \text{lose}\}} -p_{er} \log p_{er}
  \]
  where \(p_{er}\) is the proportion of evaluation results for each outcome (win, tie, lose) across 2k evaluations.

### Aggregation Strategy
- **Per-Sample**: Calculate scores for each response in both positions.
- **Per-Condition**: Average scores across multiple evidences and positions.
- **Overall**: Use BPDE to determine examples needing human intervention and integrate human annotations for final results.

## Implementation Notes

- **Prompt Template**: Ensure the evaluation prompt includes explicit instructions to avoid bias based on response order.
- **Temperature Setting**: Use a temperature of 1 for sampling diverse evaluation evidences in MEC.
- **Order of Operations**: Perform MEC before BPC to ensure multiple evidences are considered before averaging scores across positions.
