## Experimental Design

### 1. Automatic Evaluation of Language Composition
- **Procedure**: 
  - Use the FastText language identification model to classify each line in the datasets as English or non-English.
  - Set a confidence threshold of 0.6 for classifying a line as non-English.
  - For the C4.EN dataset, subsample the first 50 million examples (approximately 14% of the dataset) due to its large size. Classify the entirety of all other datasets.
- **What to Record**: 
  - Record the number of non-English tokens identified in each dataset.
  - Calculate the percentage of non-English text in each dataset.
- **Correspondence**: Figure 1, Table 3, Section 2.1

### 2. Qualitative Analysis of Non-English Texts
- **Procedure**: 
  - Randomly select 200 lines classified as non-English from each dataset.
  - Manually code each line into one of six categories: NE (non-English), BiL (bilingual), Trans. (translations), Ent. (non-English entities), En. (misclassified English), and XX (no natural language).
- **What to Record**: 
  - Record the distribution of lines across the six categories for each dataset.
  - Note examples of each type of non-English data.
- **Correspondence**: Table 1, Section 2.2

### 3. Non-English MLM Evaluation
- **Procedure**: 
  - Evaluate the perplexity of English pretrained masked language models (MLMs) on non-English text using the Wiki-40B dataset.
  - Report bits per character (BPC) to allow comparison between models with different tokenizations.
- **What to Record**: 
  - Record BPC scores for each language in the Wiki-40B dataset.
- **Correspondence**: Figure 2a, Section 3.1

### 4. POS Performance Across Languages
- **Probing Procedure**: 
  - Train a linear classifier to predict POS tags from the final layer of each model's encoder.
  - Use the average representation of subword tokens for words tokenized into multiple units.
- **Fine-tuning Procedure**: 
  - Fine-tune a subset of models for non-English POS tagging using Universal Dependencies (UD) treebanks.
- **What to Record**: 
  - Record accuracy scores for POS tagging across languages.
  - Compare performance between monolingual and multilingual models.

## Evaluation Details

- **Metric Definitions**:
  - **Percentage of Non-English Text**: Calculated as the number of non-English tokens divided by the total number of tokens in the dataset.
  - **Bits Per Character (BPC)**: A measure of language model perplexity, calculated as the average number of bits needed to encode each character in the text.
  - **POS Tagging Accuracy**: The proportion of correctly predicted POS tags out of the total number of tags.

- **Normalization and Pre-processing**:
  - For BPC, ensure consistent tokenization across models by using the same tokenization scheme for each language.
  - For POS tagging, average subword token representations for words split into multiple tokens.

- **Aggregation Strategy**:
  - For language composition, aggregate non-English token counts to calculate overall percentages.
  - For BPC and POS accuracy, report average scores across multiple runs and languages.
