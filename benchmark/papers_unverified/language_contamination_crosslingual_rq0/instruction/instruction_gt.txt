## Experimental Design

### 1. Automatic Evaluation of Language Composition
- **Procedure**: 
  - Use the FastText language identification model to label each line in the datasets as non-English if the confidence score exceeds 0.6.
  - For the C4.EN dataset, subsample the first 50 million examples (approximately 14% of the dataset) due to its large size. Classify the entirety of all other datasets.
- **What to Record**: 
  - Record the estimated number of non-English tokens for each dataset.
  - Present results as an estimate of non-English data in each dataset.
- **Correspondence**: 
  - Figure 1, Table 3, Section 2.1

### 2. Qualitative Analysis of Non-English Texts
- **Procedure**: 
  - Randomly select 200 lines per corpus that were predicted as non-English by the language classifier.
  - Manually code each line into one of six categories: NE (non-English), BiL (bilingual), Trans. (translations), Ent. (non-English entities), En. (misclassified English), and XX (no natural language).
- **What to Record**: 
  - Record the distribution of line types across the datasets.
  - Note any patterns or common errors in classification.
- **Correspondence**: 
  - Table 1, Section 2.2

### 3. Non-English MLM Evaluation
- **Procedure**: 
  - Measure the perplexity of English pretrained masked language models (MLMs) on non-English text using the Wiki-40B dataset.
  - Report bits per character (BPC) to allow comparison between models with different tokenizations.
- **What to Record**: 
  - Record BPC scores for each language and model.
- **Correspondence**: 
  - Figure 2a, Section 3.1

### 4. POS Performance Across Languages
- **Procedure**: 
  - Evaluate monolingual English models on non-English part-of-speech (POS) tagging tasks.
  - Conduct both probing (using a linear classifier on the final layer) and fine-tuning experiments.
- **What to Record**: 
  - Record accuracy scores for each language and model.
  - Report average performance across five runs with different random seeds.
- **Correspondence**:

## Evaluation Details

- **Metric Definitions**:
  - **Bits Per Character (BPC)**: Calculated as the log base 2 of the perplexity per character.
  - **POS Tagging Accuracy**: The percentage of correctly predicted POS tags out of the total number of tags.

- **Normalization and Pre-processing**:
  - For MLM evaluation, ensure text is tokenized according to each model's requirements.
  - For POS tagging, average the representation of all tokens for words tokenized into multiple subword units.

- **Aggregation Strategy**:
  - For BPC, report the average across all characters in the dataset.
  - For POS tagging, report the average accuracy across all languages and runs.

## Implementation Notes

- **Pitfalls and Caveats**:
  - Language detection is imperfect, especially for low-resource languages, which may lead to misclassification.
  - High UNK token rates in tokenization can affect downstream task performance and BPC scores.

- **Special Prompt Templates and Output Parsing**:
  - Ensure consistent use of the FastText model's confidence threshold (0.6) for language classification.
  - Use manual coding categories consistently for qualitative analysis.

- **Order of Operations**:
  - Perform automatic language identification before qualitative analysis to ensure accurate sampling of non-English lines.
  - Conduct MLM evaluation and POS tagging experiments separately to avoid confounding results.
