## Evaluation Details

- **Metric Definitions**:
  - **Accuracy**: Proportion of correctly predicted sequences. For a sequence of length L, the entire sequence must be correct to score 1.
  - **Token Edit Distance**: Number of token-level edits (insertions, deletions, substitutions) required to transform the predicted sequence into the target sequence.
  - **Brier Score**: Mean squared error between the predicted probability and the actual outcome for binary classification tasks.

- **Normalization and Pre-processing**:
  - For Accuracy, ensure that predictions are compared against the exact target sequence.
  - For Token Edit Distance, calculate the number of edits required for each prediction compared to the target.

- **Aggregation Strategy**:
  - Compute per-sample scores for each metric.
  - Aggregate scores across all samples to obtain per-condition scores.
  - Compare aggregated scores across different model scales and metrics to assess the presence or absence of emergent abilities.

## Implementation Notes

- **Pitfalls and Caveats**:
  - Ensure that the increased test dataset size is sufficient to reveal continuous improvements in performance.
  - Be cautious of the independence assumption in token-level predictions, which may not hold in practice.

- **Special Prompt Templates**:
  - Use consistent prompting strategies for arithmetic tasks to ensure comparability across different model scales and metrics.

- **Order of Operations**:
  - First, collect model outputs using the original metrics.
  - Then, change the metrics and re-evaluate the same outputs to observe changes in perceived emergent abilities.
