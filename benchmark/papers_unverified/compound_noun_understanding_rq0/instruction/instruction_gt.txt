## Experimental Design

### 1. Baseline Evaluation of VLMs on Compun
- **Procedure**: 
  - For each compound noun (CN) in the Compun dataset, generate a text prompt in the format "A photo of a {compound noun}".
  - Use each vision-language model (CLIP, OpenCLIP, ALIGN, ALBEF, BLIP, MetaCLIP) to perform zero-shot text-to-image retrieval.
  - Each CN instance includes one correct image and two distractor images depicting the constituent nouns.
  - Evaluate the model's ability to select the correct image among the distractors.
- **What to Record**: 
  - Record the accuracy of each model in selecting the correct image for each CN.
  - Collect cosine similarity scores between the text prompt and each image.
- **Correspondence**: 
  - Results are presented in Table 1 and discussed in Section 5.

### 2. Retrieval with Example Captions
- **Procedure**: 
  - For each CN, use GPT-4 to generate 5 diverse captions, each describing a scene with the CN as an object.
  - Construct prompts for the VLMs using the format: "A photo of a {class name}. An example of {compound name} in an image is {caption}".
  - Calculate the mean cosine similarity of each image with the 5 generated prompts.
  - Select the image with the highest mean similarity as the retrieval result.
- **What to Record**: 
  - Record the accuracy of retrieval using the example captions.
  - Compare the performance with baseline template-based retrieval.
- **Correspondence**: 
  - Results are shown in Table 1 and illustrated in Figure 1.

### 3. Analysis of CN Types
- **Procedure**: 
  - Categorize CNs into three types: Either Noun, Both Nouns, and None.
  - Evaluate the performance of CLIP on each category separately.
- **What to Record**: 
  - Count of misclassified instances for each CN type.
  - Average similarity scores for correct predictions in each category.

## Evaluation Details

- **Metric Definition**: 
  - Use a binary accuracy metric where a correct retrieval is scored as 1 if the cosine similarity of the prompt with the correct image is greater than with either distractor image, otherwise score as 0.
  - Formula: \( f(C, P, N1, N2) = 1 \) if \( s(C, P) > s(C, N1) \) and \( s(C, P) > s(C, N2) \), else 0.
- **Cosine Similarity Calculation**: 
  - Use the standard cosine similarity function provided by the model's library to compute similarity scores between text prompts and images.
- **Aggregation Strategy**: 
  - Compute per-sample accuracy, then aggregate to obtain per-model accuracy on the entire Compun benchmark.

## Implementation Notes

- **Prompt Generation for Captions**: 
  - Use GPT-4 with a specific prompt to generate diverse captions: "Return a list of 5 diverse captions with a compound_noun in a photo. The captions should be a maximum of 10 words and one-liners."
- **Order of Operations**: 
  - Ensure that the generation of captions and construction of prompts precedes the retrieval task.
- **Pitfalls**: 
  - Be cautious of CNs with multiple interpretations; use the most commonly known interpretation as determined by MTurk studies.
