## Experimental Design

### 1. Text-to-Image Retrieval with Compound Nouns
- **Procedure**: 
  - For each compound noun (CN) in the Compun benchmark, generate a text prompt in the format "A photo of a {compound noun}".
  - Use the VLMs to perform zero-shot text-to-image retrieval, selecting the correct image depicting the CN from a set of three images: one positive image showing the CN and two distractor images showing the constituent nouns.
  - Repeat the retrieval task using the proposed method, which involves generating five diverse captions for each CN using GPT-4. Construct prompts by appending each caption to the base prompt: "A photo of a {class name}. An example of {compound name} in an image is {caption}".
  - Calculate the mean cosine similarity between the image and the five prompts, and select the image with the highest mean similarity.
- **What to Record**: 
  - Raw outputs of the retrieval task (selected images).
  - Cosine similarity scores for each image-prompt pair.
  - Accuracy of image selection for each CN.

## Evaluation Details

- **Metric Definition**: 
  - Use a binary accuracy metric for evaluation. For each CN, the retrieval is correct if the cosine similarity of the CN image with the prompt is greater than that of both distractor images.
  - Formula: 
    \[
    f(C, P, N1, N2) = 
    \begin{cases} 
    1, & \text{if } s(C, P) > s(C, N1) \text{ and } s(C, P) > s(C, N2) \\
    0, & \text{otherwise}
    \end{cases}
    \]
  - Where \( s(.) \) denotes cosine similarity.
- **Normalization/Pre-processing**: 
  - Ensure all text prompts are formatted consistently.
  - Normalize cosine similarity scores to ensure comparability across different prompts.
- **Aggregation Strategy**: 
  - Compute per-sample accuracy for each CN.
  - Aggregate results to obtain per-condition accuracy (e.g., baseline vs. proposed method).
  - Report overall accuracy across all CNs in the benchmark.
