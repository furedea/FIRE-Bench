2) Prompting strategies comparison (CoT, Self-Probing, Multi-Step, Top-K) vs vanilla
- Procedure:

## Evaluation Details

- Answer normalization and equivalence:
  - Multi-choice (MMLU subsets): extract a single uppercase letter among provided options (A–E depending on dataset; most are A–D). Normalize by stripping spaces, punctuation, and surrounding text.
  - Yes/No (StrategyQA): accept case-insensitive variants {"yes","no"}. Map to canonical {"yes","no"}.
  - Open-number (GSM8K, SVAMP): extract the first signed/unsigned number token; accept integers or decimals; strip commas; for equality, compare as floats with absolute tolerance 1e-6. If multiple numbers appear, prefer the one immediately after "Answer" or the last number in the line containing "Answer and Confidence".
  - Dates (DateUnd): parse using a robust date parser (e.g., python-dateutil). Normalize to YYYY-MM-DD; handle day-of-week by mapping common names/abbreviations to canonical form.
  - Object Counting: integer comparison as above.
  - If no parseable answer, mark the sample as invalid; retry once with a repair prompt requesting strictly the specified format; if still invalid, drop the sample and log.

- Confidence extraction and normalization:
  - Vanilla/CoT: parse the numeric value after the final "Answer and Confidence (0-100):" line; strip "%"; clamp to [0,100]; divide by 100 to get [0,1] for scoring.
  - Self-Probing: parse "Confidence: [number]" (no "%"); clamp [0,100]; divide by 100.
  - Multi-Step:
    - Stated overall confidence: parse final line; [0,100] → [0,1].
    - Product-of-steps confidence: multiply per-step confidences (each parsed as % and converted to [0,1]); final ∏ Ci ∈ [0,1].
    - Report both; by default, use the product version for calibration/failure-prediction unless otherwise specified.
  - Top-K:
    - Parse each Gi and Pi; clamp Pi ∈ [0,100]; divide by 100.
    - For Task 2 and Table 10, use (G1, P1) as (prediction, confidence).
  - Missing confidence: If the required confidence token is missing, retry once with a repair prompt requesting only the confidence in the specified format; otherwise discard and log.

- Consistency aggregation (Eq. 1):
  - Given M responses, define the set of unique normalized answers A.
  - For each a ∈ A: Cconsistency(a) = (1/M) Σ_i I{Yi == a}.
  - Final answer â = argmax_a Cconsistency(a). Confidence for â is Cconsistency(â).
  - Ties: break by choosing the candidate with higher average verbalized confidence among tied answers; if still tied, choose lexicographically smallest.

- Avg-Conf aggregation (Eq. 2):
  - Convert each response’s confidence Ci to [0,1].
  - For each a ∈ A:
    - Numerator = Σ_i I{Yi == a} × Ci.
    - Denominator = Σ_i Ci + ε, with ε=1e-8 to avoid divide-by-zero.
    - Cavg-conf(a) = Numerator / Denominator.
  - Final â = argmax_a Cavg-conf(a); report Cavg-conf(â).

- Pair-Rank aggregation (Eq. 4; Bradley–Terry-style optimization):
  - Inputs: N independent Top-K generations S(i) = (S(i)_1, ..., S(i)_K), i=1..N.
  - Build A = unique normalized answers across all S(i).
  - For each ordered pair (u,v) ∈ A×A, u≠v, and each i:
    - If at least one of {u,v} appears in S(i), define I_i[u ≻ v] = 1 if u appears before v in S(i), else 0 (if only one appears, the other is treated as appearing after the observed one).
  - Parameterization: logits θ_u ∈ R for each u ∈ A; define P(u) = softmax(θ)_u.
  - Loss (to minimize): L(θ) = − Σ_i Σ_{u≠v} I_i[u ≻ v] · log( P(u) / (P(u)+P(v)) ).
  - Optimization:
    - Initialize θ_u = 0 for all u.
    - Use gradient descent/Adam (lr=0.1), 1000 steps or until |ΔL| < 1e-6. Add L2 regularization λ||θ||^2 with λ=1e-4 to stabilize [uncertain].
  - Output: P(u) for all u; final â = argmax_u P(u); confidence = P(â).
  - Numerical stability: compute log P(u)/(P(u)+P(v)) via logsumexp on (θ_u, θ_v).

- Expected Calibration Error (ECE):
  - Let B be the number of confidence bins [uncertain] (use B=10 as default). Define bins I_b = ( (b−1)/B, b/B ] for b=1..B.
  - For each bin b, let S_b be samples with confidence in I_b; acc(b) = mean correctness over S_b; conf(b) = mean confidence over S_b.
  - ECE = Σ_{b=1}^B (|S_b|/n) · |acc(b) − conf(b)|.
  - Report both raw (0–1) and as ×100 for table comparability when needed; always store raw.

- Failure prediction metrics:
  - AUROC: sklearn.metrics.roc_auc_score(y_true, y_score), where y_true = 1 for correct, 0 for incorrect; y_score = confidence in correctness (in [0,1]). If all scores are identical, AUROC=0.5 by convention.
  - AUPRC-Positive (PR-P): average_precision_score(y_pos, y_score), where y_pos = 1 for incorrect samples (treating errors as positive class), 0 otherwise.
  - AUPRC-Negative (PR-N): average_precision_score(y_neg, y_score), where y_neg = 1 for correct samples, 0 otherwise.
  - For Top-K/Pair-Rank, use the aggregated confidence for â as y_score.

- Reliability diagrams and confidence histograms:
  - Reliability: same bins as ECE; plot conf(b) vs acc(b) with bin sizes annotated; store (conf(b), acc(b), |S_b|).
  - Histograms: 5% bins at {50,55,...,100} (as in paper visuals); store counts per bin and fraction incorrect within each bin.

- Aggregation of results:
  - Per-sample: store prediction, confidence, correctness, and any intermediate details (e.g., per-step confidences, Top-K lists).
  - Per-condition (dataset × model × method): compute Accuracy, ECE, AUROC, PR-P, PR-N.
  - Overall: mean across datasets; also report standard deviation/variance when multiple runs (e.g., across M or seeds) are involved.

- Output parsing robustness:
  - Use regex to capture:
    - Answer and Confidence line: r"Answer and Confidence.*?:\s*([^,]+)\s*,\s*([0-9]+(?:\.[0-9]+)?)\s*%".
    - Confidence only (Self-Probing): r"Confidence:\s*([0-9]+(?:\.[0-9]+)?)".
    - Top-K: lines starting with "G[0-9]+:" and "P[0-9]+:"; tolerate minor whitespace; ignore extra commentary but prefer strict matches.
  - Strip quotes, punctuation; normalize whitespace; enforce single prediction per sample after normalization.


## Implementation Notes

- Prompts and required formats:
  - Always append: "Note: The confidence indicates how likely you think your answer is true." to every prompt (as specified in Table 1).
  - Enforce exact output formats to ease parsing; if the model deviates, send one repair prompt: "Only output in the specified format. Do not include any other words."
  - For Top-K, require exactly K lines of Gi/Pi pairs; if fewer appear, accept the present pairs; if more appear, take the first K.

- Order of operations and independence:
  - Self-Probing must use two independent chats: do not include prior turns from Phase A; pass only the question and the "Possible Answer" string into Phase B.
  - For sampling (Self-Random / Misleading / Prompting) ensure each of the M queries is an independent API call with no shared chat history. Set temperature to 0.7, top_p=[uncertain] (default 1.0).
  - Evaluate single-shot baselines (vanilla, CoT, Top-K M=1) before sampling-based methods.

- Misleading hints content:
  - Insert the hint on its own line immediately before the question text: "Hint: ...".
  - For generated misleading answers:
    - Multi-choice: random option from the provided choices.
    - Yes/No: random choice "Yes" or "No".
    - Numeric/date: draw from simple priors; ensure valid formatting (integer for counts; valid date string or weekday token).
  - Rotate through Weak Claim / Strong Claim / External Source phrasing; for group experiments, restrict to the selected group only.

- Pair-Rank implementation caveats:
  - Treat "without replacement" ranking as in the paper; in practice, deduplicate repeated guesses within a single Top-K list, preserving first occurrence.
  - If a pair (u,v) never co-occurs across all N generations, exclude it from the loss; ensure at least one comparison per candidate or add a small prior (e.g., θ_u ~ N(0, 0.1^2)).
  - Convergence checks: stop if relative loss change < 1e-6 or after 1000 steps; clip logits to [-15, 15] to prevent numerical overflow.

- Multi-Step confidence:
  - Compute both the stated "Overall Confidence" and the product-of-steps confidence; use the product version for primary scoring (as per §3.2 formula C_multi-step = Π_i Ci).
  - If any step confidence is missing or unparsable, fall back to the stated overall confidence for that sample; log the event.

- Equality checks and canonicalization:
  - Numbers: compare as floats with absolute tolerance 1e-6; treat "1" and "1.0" as equal.
  - Dates: canonicalize to YYYY-MM-DD or a canonical weekday token before equality.
  - Yes/No: canonicalize to lowercase "yes"/"no".
  - Multi-choice: ensure the option letter is actually present among the provided options; if the model outputs an option text, map it back to the letter by exact match; if ambiguous, mark invalid.

- Sampling budget and scheduling:
  - To respect API budgets, cap per-dataset sample sizes (e.g., 200–500 per dataset per strategy per model) and prioritize:
    - Core baselines (vanilla, CoT, Top-K M=1).
    - Sampling with M=5 + Consistency.
    - Aggregation study (Top-K + Self-Random + {Consistency, Avg-Conf, Pair-Rank}).
  - Run ablations (role-play, hint groups, M-scaling) on a subset of datasets representative of each task type to save budget.

- Error handling:
  - If AUROC cannot be computed due to all-equal confidences, assign AUROC=0.5 and log.
  - If ECE bins are empty, skip their contribution (weight=0).
  - For Top-K where P1..Pk do not sum to 100, accept as-is; do not renormalize for the M=1 Top-K experiment.

- Reproducibility:
  - Fix random seeds for hint selection and paraphrase generation per run.
  - Log all prompts and raw responses verbatim, along with parsed fields and any repair attempts.

- Output files:
  - Per experiment: CSV with columns [id, dataset, method, prompt_variant, sampling_variant, aggregator, M, K, prediction, confidence, correct, raw_output_json, extras_json].
  - Per condition: JSON summary with metrics (Accuracy, ECE, AUROC, PR-P, PR-N), sample counts, and configuration (temperature, bins=B, etc.).
