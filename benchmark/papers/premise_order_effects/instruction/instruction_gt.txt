## Evaluation Details

- Logical Reasoning: proof validity and accuracy
  - Normalization and parsing
    - Strip whitespace, normalize quotes, lowercase only for control phrases (“cannot be proved” etc.) but keep case-sensitive symbols for facts/rules.
    - Extract steps as lines starting with “Step” or numbered list markers. Fallback: any line containing brackets “[...]” with premise IDs.
    - Premise references: parse integers within square brackets as IDs. Allow multiple IDs per step.
    - Derived fact extraction: the trailing atom on each step after “then”/“therefore”/“derive”/“we get” is treated as the asserted atom for the step; also accept “X is true”.
  - Proof checker (deterministic)
    - Let R_rel be the set of relevant rules as (antecedent set → consequent). Let F0 be initial facts.
    - Maintain a derived set D initialized to F0.
    - For each parsed step t in order:
      - Validate all cited premise IDs exist and refer to either a Fact or a Rule supplied in the prompt. If any cited ID does not exist → rule hallucination.
      - If the step cites rules, verify that for each cited rule r = (A → B), A ⊆ D at this step. If not, and the derived fact is not already in D via any valid rule application, mark as fact hallucination.
      - If a derived fact Ft is produced, check that there exists at least one rule r in R_rel with antecedents A ⊆ D that entails Ft at this step. If none, mark as fact hallucination.
      - If the step cites only facts and claims a new fact beyond D, mark as fact hallucination.
      - Add valid Ft to D.
    - Success condition: after the final step, Goal G ∈ D. Additionally, no step triggered hallucination; otherwise the sample is incorrect.
    - If the model states impossibility (detect phrases such as “cannot be proved,” “not provable,” “insufficient information,” “cannot derive,” or explicitly claims the goal is false) and G ∉ D at the end → wrong refutation.
    - If the model never reaches G ∈ D and did not match wrong-refutation phrases, classify as fact hallucination if any invalid fact was introduced; else classify as rule hallucination if any invalid rule citation occurred; else classify as wrong refutation if it terminated claiming failure; else classify as [uncertain]/other (do not include in Table 1; only needed if performing extended error analysis).
  - Accuracy
    - Per-variant accuracy: 1 if proof checker accepts; 0 otherwise.
    - For each base problem and distractor level, compute accuracy per τ ∈ {1.0, 0.5, 0.0, -0.5, -1.0}.
    - Shuffled accuracy = mean over τ ∈ {0.5, 0.0, -0.5}.
    - Aggregate by:
      - Per-m (number of relevant rules) averages.
      - With vs. without distractors (k ∈ {0,5,10}).
      - Per-τ curves (Figures 5 and 6).
    - Error analysis (Table 1): restrict to m=12, k=0; report proportions over outcomes in {Correct, Wrong refutation, Rule hallucination, Fact hallucination}.

- Kendall’s tau computation and order generation
  - Given forward order π (length m) and a candidate order σ, compute Kendall’s tau correlation:
    - τ = (C − D) / (m·(m−1)/2), where C is the number of concordant pairs, D discordant pairs.
    - Use exact computation with no ties.
  - For target τ* ∈ {0.5, 0.0, −0.5}:
    - Sample random permutations σ until |τ(σ, π) − τ*| ≤ 0.05; cap at 10,000 trials. If not found, relax to 0.10. Record actual τ achieved for reporting.
  - Backward order uses σ = reverse(π) which yields τ = −1.0 by construction. Forward uses σ = π (τ=1.0).

- R-GSM scoring
  - Output normalization
    - Extract the final numeric answer using regex capturing the last occurrence of:
      - Integers: [-+]?[0-9][0-9,]* (remove commas).
      - Decimals: [-+]?[0-9]*\.[0-9]+
      - Fractions in plain text (e.g., “3/4”): treat as float numerator/denominator.
    - Prefer the last numeric token in the output; if multiple, use the final one.
  - Matching
    - Convert ground-truth answer string to numeric form (strip formatting).
    - Correct if:
      - Exact integer equality; or
      - Absolute difference ≤ 1e-6 for decimals; or
      - For percentage-form in output (e.g., “25%”) when ground-truth is 0.25, treat as correct if within 1e-6 after scaling by 100 or 1, whichever matches best.
  - Aggregation
    - Overall accuracy on all originals and on all reordereds (Table 2a).
    - Subset accuracy (Table 2b): restrict to problems solved correctly in original; recompute accuracy on the paired reordered only.
    - Breakdowns:
      - Reasoning steps: approximate from GSM8K reference solution text by counting arithmetic operators. Define step count as the number of occurrences of any of {+, -, *, /} tokens or their word forms {"plus","minus","times","multiplied","divided","difference","sum","product","quotient"}; collapse repeated operators in the same sentence to 1 step; bin as in Table 4a categories {2,3,4,5,6,7,8}.
      - Sentences: count in the problem statement; bin as {5,6,7,8}.

- Decoding parameters (all tasks)
  - Temperature = 0, top_p = 1.0, n = 1, stop = default [uncertain], max_tokens = [uncertain].
  - No system or few-shot exemplars; pure zero-shot (except the explicit derivation instruction for logical reasoning).


## Implementation Notes

- Prompt template LR-0 (logical reasoning; zero-shot, derivation required)
  - Use exactly:
    - Facts:
      - “Facts:” then enumerate as “Fact 1: a1 is true.”, …
    - Rules:
      - “Rules:” enumerate as “Rule 1: If a1 and a2, then b1.” (use “and” for 2–3 antecedents; for arity 1 omit “and”).
    - Goal:
      - “Goal: g is true.”
    - Instruction:
      - “Provide a step-by-step derivation to prove the Goal using only the numbered Facts and Rules. At each step, cite the premise indices in square brackets (e.g., [Rule 3], [Fact 2]) and state the new fact you derive. Do not invent any new facts or rules. End with a final line: ‘Therefore, g is true.’”
  - Output parsing expects lines starting with “Step 1: …”, but the checker should accept any numbered list; rely on square-bracketed citations.

- Rule/fact naming
  - Use short lowercase atom names (e.g., a1, b2, c3) to minimize token length and reduce ambiguity in string matching. Derived facts must exactly match these atom names (with “is true” suffix).

- Interleaving distractors
  - After obtaining the relevant-rule order σ for a given τ, create the final Rules section by interleaving distractors at approximately uniform intervals:
    - If there are k distractors and m relevant rules, place a distractor after every ⌈m/(k+1)⌉ relevant rules (round-robin). Keep distractor order fixed across τ for that base problem.

- Graph generation for base problems
  - Construct a DAG of implications that yields a linearizable forward proof order of length m:
    - Sample arities uniformly from {1,2,3} with at least 1 rule of arity ≥2 if m ≥ 6 [recommended].
    - Ensure acyclicity; use topological construction:
      - Initialize D = F0 with 2–4 seed facts.
      - Iteratively add a new rule whose antecedents are sampled from D (size equals chosen arity) and whose consequent is a new atom not in D; append the consequent to D.
      - The final consequent is the Goal.
  - Distractors must not be triggered by any subset of D; ensure their antecedents contain at least one atom never derivable in the graph.

- τ category feasibility
  - For small m, τ values are discrete; exact τ=±0.5 may be impossible. Use the acceptance bands specified in Evaluation Details; always log the realized τ and include it in analysis plots/tables.

- Error categorization precedence (Table 1 replication)
  - If both hallucination and refutation cues appear, assign wrong refutation only if the model explicitly claims impossibility; else assign hallucination type based on the first detected violation in step order (rule → fact).
  - For Table 1, run the classifier strictly on m=12, k=0, all τ; do not mix with other settings.

- R-GSM reconstruction caveats
  - Keep the final question sentence unchanged.
  - Avoid changing causal/temporal logic; if a permutation alters the timeline or prerequisites such that the gold answer is no longer valid, discard and try another permutation.
  - Minor edits allowed: pronouns, articles, tense to maintain grammaticality; never change numerals or relational comparators.
  - Human verification is required to confirm the gold answer is preserved. Record a justification string and the exact permutation indices for audit.

- Order of operations
  - Logical reasoning:
    1) Run k=0 across τ to get Figures 3 and 5 (or their tables).
    2) Then k=5 and k=10 for Figures 4 and 6.
    3) Finally, compute the m=12, k=0 error analysis (Table 1).
  - Position ablation (Appendix D) after the above.
  - R-GSM:
    1) Evaluate originals.
    2) Evaluate reordereds.
    3) Compute overall, then the init-100% subset, then breakdowns.
