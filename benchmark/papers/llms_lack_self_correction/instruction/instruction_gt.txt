## Evaluation Details

- Output normalization and parsing
  - Math:
    - Extract the last occurrence of \boxed{...}. Regex: r"\\boxed\{([^}]*)\}". If none, fallback to the last numeric token in the final line.
    - Normalize: strip whitespace; remove currency symbols and commas; if float-like, parse to Decimal then to canonical string without trailing zeros (e.g., 75.00 → 75); keep sign.
    - Invalid or unparsable → mark as incorrect.
  - Commonsense:
    - Extract the last parenthetical single letter A–E. Regex: r"\(([A-E])\)(?!.*\([A-E]\))".
    - Uppercase; if multiple or none, try to infer from “final answer” line; else mark as incorrect.
  - Multi-hop:
    - Extract the final short answer from the last line after “Final Answer:” (case-insensitive). If absent, take the final line trimmed.
    - Normalize using HotpotQA’s standard normalization: lowercase, remove punctuation, articles (a/an/the), extra whitespace. If the official script is not available, implement SQuAD-style normalization identically.
- Exact Match (EM)
  - Per-sample EM = 1 if normalized prediction equals normalized gold; else 0.
- Aggregation
  - Per-condition accuracy = mean(EM) over the sampled set.
  - Report 95% CIs via normal approximation or bootstrap (1,000 resamples; seed 20231014) if desired.
  - Answer-change categories (for intrinsic self-correction after round 2):
    - Let y0 = correctness of initial; y2 = correctness of final after round 2; and s0, s2 = normalized extracted answers. Category:
      - No Change: s2 == s0 (string equality after normalization).
      - Correct⇒Incorrect: y0=1 and y2=0 and s2 != s0.
      - Incorrect⇒Correct: y0=0 and y2=1 and s2 != s0.
      - Incorrect⇒Incorrect: y0=0 and y2=0 and s2 != s0.
    - Compute percentages over evaluable samples; report counts and proportions.
- Oracle stopping (Task 2)
  - Use the same EM function to gate steps:
    - If EM(initial)=1 → stop.
    - Else run feedback+revise; if EM(revise1)=1 → stop; else run the second feedback+revise.
  - Record realized calls per item and average calls per dataset.
- Self-consistency (Task 4)
  - Generate N independent samples with identical initial prompt.
  - Parse/normalize each sample’s final answer as above.
  - Majority voting over normalized answers; ties broken deterministically by lexicographic order of normalized answers (record as tie).
- Multi-agent debate (Task 4)
  - Parse each agent’s final stated answer per round; use the debate protocol’s final selection (judge or consensus) [uncertain].
  - EM computed on the selected final answer.
- Concept coverage (Task 5) [optional]
  - For each concept c:
    - Lowercase and tokenize on whitespace/punctuation.
    - If multi-word: require all tokens to appear as whole-word matches in the generated text (lowercased), in any order; for single-word: whole-word match.
  - Coverage = (# concepts matched) / (# concepts provided).
  - Average coverage over the set.

## Implementation Notes

- Pitfalls and caveats
  - Do not use labels to influence intrinsic self-correction prompts or stopping; labels are used only for evaluation (and for oracle stopping in Task 2).
  - Enforce output-format constraints in prompts; otherwise parsing failures inflate error rates.
  - For multi-hop closed-book, ensure no retrieval or external tools are used.
  - Maintain identical decoding configuration across conditions within each model to ensure fair comparison.
- Prompt templates and delimiters
  - Include explicit delimiters when passing previous answers to the feedback/refine steps:
    - Prefix the model’s prior answer with: “Previous answer:\n{model_output}”
    - Then append the feedback or refine instruction as specified.
  - For multi-agent debate, clearly mark agent roles, turns, and the final selection step; store per-agent transcripts with unique IDs [exact template text is [uncertain]].
- Output parsing logic
  - Always run extraction on the last assistant message in the step.
  - Prefer the most constrained final-answer pattern (e.g., \boxed{...} or (A)–(E)) over any earlier mentions.
  - Strip trailing punctuation from extracted spans (e.g., periods after numbers or letters).
- Order of operations
  - For each sampled item:
    1) Run initial (standard) prompting and score it (serves as baseline).
    2) Run intrinsic self-correction Round 1 and Round 2 sequentially and score each.
    3) For ablations, repeat 2) with the alternate feedback prompts (on disjoint sampled subsets).
    4) Separately, run oracle-stopping self-correction (Task 2) on its designated sampled subset.
    5) For math, run self-consistency and multi-agent debate on their designated subsets.
- Reproducibility
  - Fix RNG seeds for sampling and any tie-breaking.
  - Log prompts verbatim, model identifiers, timestamps, and token usage per call.
  - Preserve full transcripts for auditability (initial, feedback, refine across rounds; all agents/rounds in debate).
