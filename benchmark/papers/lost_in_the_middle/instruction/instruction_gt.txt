## Experimental Design

1. Multi-Doc QA: Position Sweep (k=20)
- Procedure:

## Evaluation Details

- Normalization and preprocessing:
  - Unicode: NFKD normalize, strip accents; lowercase.
  - Remove leading/trailing whitespace and surrounding quotes/backticks.
  - Strip punctuation characters [!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~].
  - Collapse multiple whitespace to single space; trim.
  - Remove English articles a, an, the when computing EM/containment (apply word-boundary-aware removal).
  - For QA only: Generate a set of gold aliases by normalizing each reference string using the same pipeline.

- Metrics:
  - Exact Match (EM) for QA (required):
    - For each prediction p and gold alias set G, compute:
      - em = 1 if normalize(p) == g for any g ∈ G; else 0.
  - Contains-Any (paper accuracy) for QA:
    - contains = 1 if any g ∈ G appears as a contiguous substring in normalize(p); else 0.
    - Note: Matches must be evaluated on the fully normalized strings as above.
  - KV Retrieval Accuracy:
    - For prediction p and target value v:
      - acc = 1 if normalize(p) contains normalize(v) as a contiguous substring; else 0.
    - Edge cases: If the model echoes the JSON or multiple UUIDs, count correct only if the true value appears anywhere in normalize(p).

- Aggregation:
  - Per-sample: compute metric(s) as above.
  - Per-position: average over the 50 paired samples (or the defined per-position count).
  - Overall per-condition: report series over positions; additionally record:
    - best = max over positions, worst = min over positions, gap = best − worst.
  - Also report closed-book and oracle single-point scores for the same question set as Task 1.
  - For randomized-distractor and query-aware variants: compute per-position curves and compare against the baseline (Task 1) for the same question-position pairs.

- Token-length screening:
  - Use the tokenizer corresponding to the model (e.g., cl100k_base) to compute prompt token length.
  - Enforce prompt_length + max_tokens ≤ model_context_limit; otherwise exclude/replace the sample in the paired set.

## Implementation Notes

- Fixed question sets across positions:
  - Use the exact same 50 questions for all 20 positions (and for each variant) to isolate positional effects. Do not resample across positions. If a sample is excluded due to token limits in any variant, exclude it consistently across all positions for that variant and replace it globally before starting generation.

- Document ordering invariants:
  - Baseline: Keep the k−1 distractors sorted by decreasing retrieval score (or original_retrieval_index) and only move the gold passage to the desired position.
  - Randomized-order variant: Uniformly shuffle the k−1 distractors with a fixed seed per question (store the permutation), then insert the gold passage at position i.
  - Do not change document contents; do not truncate passages.

- Prompt fidelity:
  - Use the exact instruction sentence provided above (including “some of which might be irrelevant”).
  - For the randomized-order variant, append exactly: The search results are ordered randomly.
  - For query-aware contextualization, place the first Question: {question} line above the documents and repeat the same question again immediately before Answer:.

- Output parsing:
  - Do not attempt to extract spans from the output; rely solely on normalization rules. EM is computed against the entire normalized output string; Contains-Any checks normalized substring inclusion.
  - For KV retrieval, UUID formats are strict; treat hex-case-insensitively (lowercase in normalization). If multiple UUIDs appear, success requires the correct value UUID to appear at least once.

- Decoding:
  - Use temperature=0 (greedy) to match the paper’s decoding assumption.
  - Disable sampling variability across runs (set seeds where possible).

- Position indices:
  - Positions are 1-indexed from the start of the context. Record and report with this convention.

- Feasibility constraints:

  - For k=30, ensure token budgets are respected; if too many exceed limits, restrict to k=20 (primary) and k=10 (secondary) as the core reproducible conditions.
