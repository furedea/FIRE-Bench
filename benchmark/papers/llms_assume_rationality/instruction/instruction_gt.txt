## Evaluation Details

- Pre-processing and normalization:
  - Dataset filtering: exclude “ambiguous” problems (no explicit probabilities). Also exclude [uncertain: problems with feedback present] to match the ≈9,831 subset.
  - Mapping to A/B: record a per-problem RNG seed (e.g., hash of problem_id) to deterministically swap options for reproducibility.
  - Forward-2 JSON outputs:
    - Accept keys "A"/"B" (case-insensitive). If not valid JSON, attempt regex extraction of two numeric percentages in order of lines mentioning Machine A then Machine B.
    - Normalize negatives to zero, clip to [0,100], then renormalize so A%+B%=100 within a tolerance of ±1. Convert to p_model(A)=A%/100.
  - Forward-1/3 parsing:
    - Extract the final letter ‘A’ or ‘B’ using a regex on the last line. If multiple occurrences, take the last explicit uppercase A/B token not within a word.
    - For CoT, prefer the last standalone ‘A’/‘B’. If ambiguous, fall back to the first uppercase A/B after a colon or “Answer:”.
  - Inverse parsing:
    - Try strict match for “Choice 1” or “Choice 2”.
    - Else accept “1” or “2” if a single-digit line.
    - If “tie” or explicit equivalence, record tie. If no parse, re-prompt once with: “Please classify your previous answer as exactly one of: Choice 1 / Choice 2 / tie.”
- Expected value computation:
  - EV(option) = sum_i q_i * r_i using the problem’s given probabilities q_i and dollar outcomes r_i. Ignore the 10% bonus and the $0 floor for negative outcomes in EV (those are prompt context only).
  - EV tie-handling: if |EV_A − EV_B| < 1e-8, treat as tie with EV_model(A)=0.5; otherwise EV_model(A)=1 if EV_A>EV_B else 0.
- Human ground truth:
  - Use per-problem empirical human P_hum(A) from the dataset. If counts are given, P_hum(A)=n_A/(n_A+n_B).
- Aggregation (forward tasks):
  - Per problem i, compute p̂_i = p_model(A).
  - Metrics:
    - Pearson r: scipy.stats.pearsonr({p̂_i}, {P_hum_i}).
    - Spearman ρ: scipy.stats.spearmanr({p̂_i}, {P_hum_i}), average-rank method for ties.
    - MSE: mean_i (p̂_i − P_hum_i)^2.
  - Correlation vs EV: use the same p̂_i vector but replace human vector with {EV_model(A)_i} ∈ {0,0.5,1}.
  - Report all metrics across the full filtered problem set.
- Aggregation (inverse tasks):
  - For each sample m:
    - Wins score: for each decision d, wins_m(d)=Σ_{d’≠d} [1{d beats d’}] + 0.5*Σ_{d’≠d} [1{tie with d’}].
    - Ranking r_m is the descending sort of wins_m; for ties, break by lower average opponent rank (computed iteratively) or leave as averaged ranks (compatible with Spearman).
  - Average ranking: r̄(d) = mean_m r_m(d). If using averaged ranks per sample, average those directly.
  - Correlations:
    - Against human ranking vector r_hum (length 47): ρ = spearmanr(r̄, r_hum).
    - Against rational models: compute ρ with absolute utility and relative utility predicted rankings (see below), as well as with “likelihood” and “marginal likelihood” rankings.
- Rational model computations for inverse tasks:
  - Forward model choice likelihood (Luce/softmax): p(c=oj | u, A) = exp(U_j) / Σ_k exp(U_k), where U_j = Σ_{x∈oj} u_x (linear additivity).
  - Posterior p(u | c, A) ∝ p(c | u, A) p(u). Prior p(u) [uncertain: use independent zero-mean Gaussian N(0, σ^2) with σ=1].
  - Absolute utility score for x: E[ u_x | c, A ] estimated by Monte Carlo over u ~ p(u|c,A) (e.g., 50k samples) [budget permitting; else 10k].
  - Relative utility score for x: p(∀i, u_x > u_i | c, A) estimated by the same posterior samples.
  - Likelihood-only score: p(c | ū, A) using a single fixed utility vector ū with ū_x = 1, ū_others = 1 (or [uncertain: zero-mean]) and ranking by that scalar (use as a simple baseline).
  - Marginal likelihood score: 1/p(c | A) estimated by Monte Carlo over u ~ p(u) (same prior as above), then invert. Rank by this “surprisal”-like score.
  - For each decision, compute the respective scalar score and rank decisions accordingly; then compute Spearman correlations with LLM r̄ and with humans.
- Statistical reporting:
  - Report two-sided p-values from scipy.stats for Pearson/Spearman.
  - Confidence intervals via bootstrap:
    - Forward tasks: resample problems with replacement (B=1,000) to get 95% CIs for r, ρ, and MSE.
    - Inverse tasks: resample samples m with replacement (B=1,000) to get 95% CIs for ρ.

  - Fix a global seed (e.g., 2025) for shuffling A/B per problem and all inverse-task randomizations (order of decisions, options within a decision, items within options, and item-name mappings).

  - Full pairwise evaluation requires 1,081 queries per sample. With M=43/42 samples per context and two prompting styles, this can be expensive.
